# 项目目标规划

## 职责范围

- 收集与固化项目目标、范围与阶段性里程碑
- 规划项目进度线并按阶段推进实现
- 明确技术栈与关键依赖的选型与落地路线
- 在目标变更时同步调整路线与更新记录

## 适用场景

- 启动新项目或新增核心目标
- 目标发生变化或范围扩展
- 进入新阶段需要调整进度线或技术栈

## 核心规则

1. 项目目标必须写入技能书并与项目基础保持一致
2. 必须形成可执行的项目进度线并明确阶段产出
3. 技术栈必须与项目目标和架构约束一致
4. 任何目标或进度线变化必须同步更新记录与文档

## 项目进度线模板

1. 目标确认：业务目标、边界与非目标
2. 需求拆分：模块清单与优先级
3. 设计阶段：数据模型、API 设计、依赖关系
4. 实现阶段：Repository、Service、Handler、Router
5. 质量阶段：测试覆盖、性能与安全验证
6. 交付阶段：部署、监控与维护计划

## AI 行为约束

- 必须优先确保目标与范围明确后再进入实现
- 不允许绕过进度线直接大范围实现
- 不允许新增目标但不更新文档与更新记录

## 更新规则

- 目标收集方式变化
- 进度线模板变化
- 技术栈规划方式变化
