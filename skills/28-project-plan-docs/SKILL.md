# 项目计划文档配套

## 职责范围

- 为每个项目计划创建需求、设计、任务三类文档
- 约束文档与项目目标规划的对齐与版本关联
- 规范文档创建、实施、进度更新与质量控制流程
- 保障计划进度线与文档状态一致

## 适用场景

- 启动新的项目计划或目标周期
- 项目目标规划新增或调整计划项
- 进入新阶段需要拆解任务或更新进度线

## 核心规则

### 1. 文档清单与内容要求

1. 每个项目计划必须同时具备需求文档、设计文档、任务文档
2. 需求文档必须包含功能需求与非功能需求
3. 设计文档必须包含系统架构、接口设计、数据库设计与关键技术方案
4. 任务文档必须包含任务分解、时间节点与负责人
5. 三份文档必须存在明确的版本关联与引用关系

### 2. 文档创建阶段

1. 项目启动时同步创建三份文档模板
2. 文档内容必须与项目目标规划的计划条目保持一致
3. 若计划变更，三份文档必须同步更新

### 3. 开发实施阶段

1. 开发必须严格依据任务文档的分解任务执行
2. 每个任务完成后必须更新任务文档状态
3. 任何偏离计划的实现必须先更新任务文档与计划说明

### 4. 进度更新阶段

1. 完成每个计划周期后更新项目目标规划的进度线
2. 进度更新必须包含实际完成情况与计划对比
3. 进度更新必须引用对应任务文档的状态

### 5. 质量控制

1. 文档需经项目负责人审核通过后方可进入开发阶段
2. 每个里程碑需进行文档与实现一致性检查
3. 最终进度更新需由项目经理确认

## AI 行为约束

- 不允许缺少任一文档就进入开发阶段
- 不允许文档与项目目标规划不一致仍推进实现
- 不允许更新进度线但不同步任务文档状态

## 更新规则

- 文档模板或内容要求变化
- 计划周期或进度线更新方式变化
- 审核与一致性检查流程变化
