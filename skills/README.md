## 🎯 使用指南

### 对于开发者

1. **入门**: 先阅读 `00-project-map` 了解项目整体结构
2. **开发**: 根据任务类型阅读对应的技能书
3. **审查**: 使用技能书作为代码审查的检查清单

### 对于 AI

1. **任务开始前**: 必须先读取 `00-project-map`
2. **执行过程中**: 根据任务类型加载对应的技能书
3. **检测违规**: 按照技能书定义的规则进行检查和处理
4. **代码变更时**: 必须检查是否需要更新 docs 文档（参考 `15-documentation`）

## 🔄 更新流程

### 触发条件

以下情况必须更新技能书：

1. 新增/删除模块 → 更新 `00-project-map` + `15-documentation`
2. 模块职责变化 → 更新 `00-project-map` + `15-documentation`
3. 依赖关系变化 → 更新 `00-project-map` + `02-design-specs` + `15-documentation`
4. 新增技术决策 → 更新 `01-project-foundation` + `15-documentation`
5. 新增规则约束 → 更新 `03-rules` + `15-documentation`
6. 新增公共组件 → 更新 `04-reusables` + `15-documentation`
7. 修改配置文件 → 更新 `15-documentation`
8. 新增/修改 API → 更新 `15-documentation`

### 更新顺序

```
00-project-map → 01-project-foundation → 02-design-specs → 03-rules → 04-reusables → 其他技能书
```

### 更新检查

- [ ] 代码变更与技能书一致
- [ ] 依赖关系正确
- [ ] 示例代码可运行
- [ ] 更新日期已修改
- [ ] 相关技能书已同步
- [ ] **docs 文档已同步更新（如果需要）**

## 📊 项目信息

- **项目名称**: Go Scaffold
- **项目类型**: Go 后端项目
- **Go 版本**: 1.24.6
- **技术栈**: 
  - Web 框架: Gin
  - ORM: GORM
  - 缓存: Redis
  - 日志: Zap
  - 认证: JWT
  - 权限: Casbin (RBAC)
  - 协程池: ants
  - 国际化: go-i18n
- **架构模式**: 分层架构 + 依赖注入容器
- **项目版本**: 0.1.2
