# 工作流库

## 职责范围

- 提供可复用的工作流模板
- 约束工作流的命名与存放位置

## 适用场景

- 新增模块或功能
- 修复高风险缺陷
- 需要跨模块协作的改动

## 核心规则

1. 可复用工作流必须放在 `skills/workflows/`
2. 每个工作流必须包含目标、步骤、验证方式
3. 工作流必须与 `99-workflows` 的总流程一致

## 可复用工作流模板

### 新增能力工作流

1. 确认模块归属与依赖方向
2. 设计数据模型与接口
3. 实现 Repository、Service、Handler
4. 注册路由并补充测试
5. 更新技能书与必要 docs

### 缺陷修复工作流

1. 复现问题并定位范围
2. 编写最小修复方案
3. 增加或更新测试
4. 验证修复与回归
5. 更新技能书与更新记录

## AI 行为约束

- 禁止创建未包含验证方式的工作流
- 禁止绕开 `99-workflows` 的必经步骤

## 更新规则

- 新增或调整工作流模板
- 变更工作流存放规则
